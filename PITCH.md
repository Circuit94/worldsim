# WorldSim — AI 交互式培训引擎

## 一句话

**用一句话生成一个有记忆的培训场景，让员工在 AI 模拟的真实环境中练手。**

WorldSim 是一个 AI 驱动的交互式培训引擎。输入一句场景描述，引擎即时生成带有记忆系统的 NPC、因果规则和自动演化事件的模拟环境。学员在其中做决策，系统自动追踪行为模式并输出能力评估报告。

---

## 问题

**企业培训（$380B 全球市场）正被三个痛点困住：**

**成本高、无法规模化。** 一次高管领导力培训动辄数万元，依赖真人教练 + 角色扮演。培训 100 人需要 100 倍资源。

**效果难以量化。** 培训结束后没人知道学员真正学到了什么。评估靠问卷自评，不是行为数据。产品经理没法回答"这次培训的 ROI 是多少"。

**场景单一、缺乏真实感。** PPT + 视频无法模拟真实工作中的动态博弈。真实场景里客户会变脸、同事会施压、突发事件会打断计划——这些靠 PPT 教不会。

---

## 方案

**WorldSim 让每个培训场景变成一个可交互的 AI 世界：**

输入 "销售谈判：面对一个预算紧张但有决策权的采购总监" → 引擎即时生成一个办公室环境，里面有一个有记忆、有态度、有自己目标的 AI 采购总监。学员的每个决策都会影响 NPC 的态度和后续行为——说错话对方会冷淡，抓住痛点对方会主动让步。

**培训结束后自动输出：**
- 决策路径回放（你在哪一步做对了/做错了）
- NPC 态度变化曲线（对方是怎么被你说服/激怒的）
- 能力维度评分（沟通力、应变力、谈判节奏感）
- 同场景多人对比（同一个客户，不同学员的表现差异）

## 聚焦策略：先做透销售谈判

**为什么选销售：** 场景标准化程度高（SPIN/Miller Heiman 等成熟方法论可对标）、效果可量化（成单率）、决策者明确（销售 VP）、竞品验证了 PMF（Second Nature $30M 融资只做销售）。

**v0.3 目标——销售谈判培训 MVP：**

| 能力 | 当前状态 | 目标状态 |
|------|----------|----------|
| 场景模板 | 通用生成 | 5 个难度递进的销售场景（初次拜访→异议处理→价格谈判→竞品对比→高管汇报） |
| 评估标准 | 通用 5 维度 | 对标 SPIN Selling 方法论的评估 rubric |
| NPC 行为 | 通用态度模型 | 销售场景专属行为树（预算异议、竞品比较、决策拖延等） |
| 输出报告 | 通用能力雷达图 | 销售漏斗各阶段表现 + 具体话术改进建议 |

**做透一个场景后再扩展。** Second Nature 只做销售拿了 $30M——不是因为他们不能做别的，是因为一个场景做透比十个场景做浅更有价值。

## 评估系统：现状与改进路径

### 当前架构（三层评分）

```
优先级 1：LLM Structured Eval Tags（JSON 格式，每回合实时评估）
优先级 2：Narrative 内嵌标签（兼容旧格式）
优先级 3：Heuristic Baseline（行为指标降级方案）
```

**Heuristic Baseline 的局限性（已知）：** 第三层降级方案基于关键词匹配 + 决策长度等行为指标。这不是精确评估，是"有总比没有好"的兜底。当 LLM eval tags 可用时（正常情况），此层不会被调用。

### 改进路线

| 阶段 | 方法 | 目标 |
|------|------|------|
| 当前 | LLM 实时评估 + heuristic fallback | 有评分，可参考 |
| Pilot 后 | 人类专家标注 20 个 session，计算与系统评分的 Cohen's Kappa | 量化一致性，找到系统性偏差 |
| v0.4 | 基于标注数据微调评估 prompt，引入 rubric-based scoring | Kappa > 0.6（中等一致性） |
| v0.5 | 多评估者交叉验证 + 企业自定义评估标准 | 可向 HR 展示信效度数据 |

**核心认知：** 评估系统的可信度不是靠技术迭代解决的，是靠数据积累解决的。所以第一步是拿到真实 session 数据。

## 为什么是现在

**LLM 成本断崖式下降。** DeepSeek / Gemini Flash 让单次 agent 推理成本降到 ¥0.01 以下。"每个学员独立跑一个完整模拟"的商业模式第一次经济可行。

**企业 AI 培训需求爆发。** 2025 年 Gartner 预测 70% 企业将引入 AI 辅助培训。市场上有 AI 视频培训（Synthesia）、有 AI 问答机器人，但没有"AI 模拟一个完整场景让你在里面做决策"的方案。

**交互式培训效果远超被动学习。** Harvard Business School 案例教学法的核心就是"把学员放进场景里逼他做决策"。WorldSim 把这个方法论数字化、规模化。

---

## 技术壁垒

不是 Prompt 套壳。架构有五层不可替代性：

**Stanford Generative Agents 架构。** 基于 Park et al. (UIST 2023) 论文实现。每个 NPC 遵循「观察 → 记忆 → 反思 → 规划 → 行动」的认知循环。NPC 会记住学员之前做了什么、会对学员产生态度变化、会主动制定自己的计划。这不是"每句话独立推理"的聊天机器人，是有持续状态的智能体。

**确定性规则引擎兜底 LLM。** LLM 输出经过 7 道硬约束校验（JSON Schema 验证、状态一致性检查、边界条件钳制、因果链验证等），防止状态腐败和不合理输出。培训系统不能因为 LLM 幻觉而给出错误评估——这是 B 端的刚需。

**极致 Token 效率。** ~200 tokens/agent/回合，成本仅为全量 context 方案的 1/20。通过滑动窗口记忆 + 紧凑 Prompt 设计实现。这决定了"每人独立跑完整模拟"的经济可行性。

**Seed Hinting + Session Diffing。** 同主题 + 同种子传给 LLM 后，生成的场景在结构上高度相似。配合完整的 session 导出和行为对比工具，支持"同场景多学员对比"的培训分析。路线图中计划引入程序化场景骨架，实现确定性复现。

**零成本视觉系统。** 17 种像素风地形贴图 + 13 种动漫风角色头像，全部编译时预生成。不依赖任何图片生成 API，零运行时视觉开销。

## 从 Demo 到产品：工程化路径

**当前架构的局限性（已知）：**
- 前端直连 LLM API，API key 存浏览器——无法通过企业 IT 安全审查
- 无后端服务、无用户系统、无持久化（除 localStorage）
- 这是 Demo 架构，不是交付架构

**v0.4 最小可交付架构：**

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────┐
│  Web Client │────▶│  API Server      │────▶│  LLM APIs   │
│  (React)    │     │  (Hono/CF Worker)│     │  (Gemini)   │
└─────────────┘     ├──────────────────┤     └─────────────┘
                    │  • API Key 代理   │
                    │  • Session 存储   │
                    │  • 用户鉴权       │
                    │  • 多租户隔离     │
                    └──────────────────┘
```

**为什么现在不做：** 在验证 PMF 之前，过早投入工程化是浪费。先用 Demo 架构跑通 pilot，确认用户价值后再投入后端建设。这是有意识的优先级选择，不是能力缺失。
