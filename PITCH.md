# WorldSim — 投资人 Pitch

## 一句话

**用一句话生成一个有记忆的世界。**

WorldSim 是一个 AI 世界模拟引擎，将自然语言转化为可交互的沙盒世界。NPC 拥有记忆、态度和自主目标，世界遵循因果规则自动演化。它不是游戏，是企业培训、行为仿真和 AI Agent 测试的底层基础设施。

---

## 问题

三个万亿级市场正在被低效方案困住：

**企业培训（$380B 全球市场）**：靠 PPT + 真人角色扮演，成本高、无法规模化、效果难以量化。一次高管领导力培训动辄数万元，但没人知道学员真正学到了什么。

**产品用户研究（$85B）**：招募真实用户做 UX 测试周期长、样本小、无法做 A/B 对比路径。产品经理想知道「如果这里改个按钮，用户会怎么走」，但答案要等两周。

**AI Agent 测试（新兴赛道）**：Agent 上线前缺少安全测试环境。直接让 Agent 面对真实用户太危险，但 mock 环境太简陋，无法复现真实场景的复杂性。

---

## 方案

一个引擎，三种商业模式：

**探索模式（C 端获客）**：玩家输入一句话，AI 即时生成可探索的世界。NPC 有态度有记忆，会根据你的行为改变对你的看法。用作产品引擎 demo 和 C 端留存验证。免费。

**培训模式（B 端核心变现）**：企业客户输入培训场景（医疗急诊、危机公关、销售谈判），AI 模拟真实职业环境，自动评估学员的决策质量，输出能力报告。定价 ¥5-50/次/人。可规模化到百万并发。

**仿真模式（平台级 SDK）**：Headless API，无需 UI。批量跑 AI Agent 行为仿真、用户路径模拟、策略 A/B 测试。输出结构化行为数据。按 API 调用计费。

---

## 为什么是现在

1. **LLM 成本断崖式下降**：DeepSeek 等模型让单次 agent 推理成本降到 ¥0.01 以下，使得「批量跑 1000 次模拟」的商业模式第一次可行
2. **企业 AI 培训需求爆发**：2025 年 Gartner 预测 70% 企业将引入 AI 辅助培训，但市场上没有真正交互式的方案
3. **Agent 生态成熟**：OpenAI/Anthropic/Google 都在推 Agent 框架，但都缺少安全测试环境

---

## 技术壁垒

不是 Prompt 套壳。我们的架构有四层不可替代性：

**Stanford Generative Agents 架构**：基于 Park et al. (UIST 2023) 论文实现。每个 NPC 遵循「观察 → 记忆 → 反思 → 规划 → 行动」的完整认知循环，不是每次从零推理。这意味着 NPC 会记住你做过什么、会对你产生态度变化、会主动制定自己的计划。

**确定性规则引擎兜底 LLM**：LLM 输出经过 7 道硬约束校验（JSON Schema、状态一致性、边界条件、因果链验证等），防止状态腐败和不合理输出。这是企业级场景的刚需——培训系统不能因为 LLM 幻觉而给出错误评估。

**极致 Token 效率**：~200 tokens/agent/回合，成本仅为全量 context 方案的 1/20。通过滑动窗口记忆 + 紧凑 Prompt 设计实现。这决定了批量仿真场景的经济可行性。

**Seed 可复现**：相同种子生成相同世界结构，支持科学对照实验和结果复现。这是学术研究和企业合规场景的硬需求。

**零成本视觉系统**：17 种像素风地形贴图 + 13 种动漫风角色头像，全部编译时预生成。不依赖任何图片生成 API，无运行时视觉开销。智能关键词匹配将 AI 输出的自然语言描述自动映射为对应视觉资源——让 B 端客户看到"像产品"而非"像 Demo"。

## 市场验证

同赛道已有公司拿到融资，验证了需求真实存在：

| 公司 | 阶段 | 做什么 | 我们的关系 |
|------|------|--------|-----------|
| Artificial Societies | YC W25 · $5.35M | AI 人格模拟用户，做产品测试 | 我们的仿真模式是同一赛道 |
| AgentHub | YC S25 | AI Agent 测试沙盒 | 我们的 Rule Engine + SDK 解决同样问题 |
| Tavus / Synthesia | $78M+ | AI 视频培训 / 数字人 | 我们是交互式培训，场景还原更强 |
| AI Dungeon / NovelAI | $200M+ revenue | AI 叙事 / 文字冒险 | 我们有记忆系统 + 结构化输出 |

这些公司验证了市场，但没有一家同时覆盖培训 + 仿真 + 探索三个场景。WorldSim 的统一引擎架构是差异化。
